## STRESS TESTING OBJECTIVES

Beyond any custom navigation, you must also test the app's resilience:

### 1. Erratic User Behavior Simulation
- Rapid tapping on various elements
- Quick back-and-forth navigation
- Scrolling while content is loading
- Tapping during animations/transitions

### 2. Edge Case Exploration
- Empty states (what happens with no content?)
- Boundary conditions in lists/feeds
- Deeply nested navigation paths
- Unusual navigation patterns

### 3. Mistake Recovery Testing
- Accidental navigation (can you easily go back?)
- Partial form completion (what happens if you leave?)
- Mid-flow abandonment recovery
- Unexpected back button behavior

### 4. Design Intent Testing
- Does the UI guide users correctly?
- Are dark patterns present? (hidden settings, confusing options)
- Is the happy path clear?
- Are escape hatches available when needed?

### 5. Breakability Testing
- Try to confuse the navigation
- Look for inconsistent states
- Find circular navigation loops
- Identify screens without clear exit

## SAFETY RULES
- AVOID: Logout, delete, purchase, send, block, unfollow actions
- AVOID: Any action that modifies user data permanently
- OK: Navigation, viewing, scrolling, opening menus
